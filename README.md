# ナリタシフト

なりた犬猫病院のパートスタッフ向け、シフト希望管理システム。

## 公開URL

https://dvv88mnj4w-hue.github.io/narishift/

## 合言葉

- 入り口（全体）：パートのシフト
- 管理者画面（追加）：水田

## 使っている技術

- HTML / CSS / JavaScript（フレームワークなし）
- Firebase（Firestore）：シフトデータ・スタッフ一覧・確定状態の保存
- GitHub Pages：サイトの公開
- 開発環境：VSCode + Live Server + GitHub

## 現在の状態（完成している機能）

- トップ画面・スタッフ画面・管理者画面
- スタッフ：シフト希望の入力・複数日選択・取消・訂正（締切15日まで）
- 管理者：全員分のカラーカレンダー表示、シフト確定・確定解除
- 管理者：締切後もスタッフの代理編集が可能
- 印刷機能（A4横向き、色分け対応）
- スタッフ：確定後は全員分の確定シフトを閲覧可能
- スタッフ：名前選択前でも、確定済みかどうかがひと目で分かる「確定お知らせ」表示
- スタッフの追加・削除（管理者のみ）
- 月送り機能（前月・翌月の表示）
- ホーム画面にアプリのように追加可能（PWA対応、Canvaで作成したアイコン使用）
- Firebaseへのデータ移行完了（シフト・スタッフ一覧・確定状態）
- 二重の合言葉によるアクセス制限（全体＋管理者画面）
- 合言葉入力画面をCanvaデザインの専用ページに変更（login.html）、日本語入力対応・表示切替ボタン付き
- Firestoreの接続方式をlong pollingに強制し、Safari（特にホーム画面アイコン起動時）の通信遅延を解消
- **Firestoreのセキュリティルールを、テストモードから「正しい構造のデータのみ許可」に変更・公開済み**（shifts/staff/confirmedの3種類以外は読み書き不可）
- パートスタッフ向け・管理者向けの導入案内お便り（PDF）を作成し配布済み
- 実際に「なりた犬猫病院」管理者へのプレゼンを実施し、試験導入の許可を得た（8月1日運用開始、9月分シフトからテスト運用予定）

## Firebaseのデータ構造

- `shifts`：シフト入力データ（年・月・日・スタッフ名ごと）
- `staff`：スタッフ一覧（名前・色・勤務パターン）
- `confirmed`：月ごとの確定状態

## 主なファイル構成

- `index.html`：トップ画面
- `staff.html`：スタッフ画面
- `manager.html`：管理者画面
- `login.html`：合言葉入力画面
- `style.css`：全体のスタイル
- `script.js`：共通処理（画面遷移・締切判定など）
- `calendar.js`：カレンダー・シフト入力・確定処理などのメインロジック
- `staffData.js`：スタッフ一覧の管理（Firebase対応）
- `firebase-init.js`：Firebase接続・データ読み書き関数（セキュリティルールと対になる構造）
- `auth.js`：合言葉によるアクセス制限
- `manifest.json`：PWA設定
- `images/`：アイコン・ヘッダー・フッター等の画像

## まだ対応していないこと（今後の課題）

- 🟡 締切日（15日）が script.js にハードコードされている。変更したい場合はコード修正が必要
- 🟡 データのバックアップ・書き出し機能（CSV等）
- 🟡 シフト確定をより確実に気づける通知（LINE通知・スマホバッジ等）は、Firebase Cloud Functionsなど本格的な追加開発が必要なため保留中
- 🟢 看護師・獣医師版（AIによるたたき台提案＋人による最終調整のハイブリッド方式を想定）
- 🟢 より厳格なアクセス制限が必要になった場合、Firebase Authentication（正式なログイン機能）の導入を検討
